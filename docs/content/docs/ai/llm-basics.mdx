## 速查卡

- **Self-Attention**：Q（Query）询问"谁相关"；K（Key）回答"我是谁"；V（Value）提供内容。Attention = softmax(QK^T/√d_k) × V
- **架构类型**：Decoder-only（GPT/Claude/LLaMA，生成，因果注意力，主流）→ Encoder-only（BERT，理解）→ Encoder-Decoder（T5/BART，翻译）
- **位置编码**：RoPE（旋转位置编码，LLaMA/GPT-NeoX 等主流模型使用）
- **Tokenization**：BPE（GPT）/ SentencePiece（LLaMA）/ WordPiece（BERT）。Token ≠ 词：英文 0.75 词/token，中文 0.5 字/token
- **推理参数**：Temperature（0=确定/代码，0.7-0.9=创意）→ Top-P（累积概率采样）→ Max Tokens（输出上限）
- **Prompt 技巧**：Zero-shot → Few-shot → CoT（逐步推理）→ 角色设定 → 结构化输出
- **模型选型**：任务类型 → 延迟 → 成本 → 隐私 → 上下文长度

## 一、Transformer 架构

### Self-Attention

核心公式：Attention = softmax(QK^T / √d_k) × V

每个 token 通过 **Q**（Query）询问「谁和我相关？」，**K**（Key）回答「我是谁？」，**V**（Value）提供实际内容。除以 √d_k 防止点积过大导致梯度消失。

### Multi-Head Attention

拆分为多个「头」，各关注不同语义维度（语法、指代、语义），最后拼接融合。

### 位置编码

Transformer 无序列概念，需注入位置信息：

| 方案 | 说明 | 代表 |
|------|------|------|
| 绝对位置编码 | sin/cos 函数 | 原始 Transformer |
| 可学习位置编码 | 训练学习 | BERT |
| **RoPE** | 旋转位置编码，通过旋转矩阵编码相对位置 | LLaMA、GPT-NeoX 等主流 |

### 架构演进

| 类型 | 代表 | 特点 |
|------|------|------|
| Encoder-only | BERT | 理解任务，双向注意力 |
| **Decoder-only** | GPT、Claude、LLaMA | 生成任务，因果注意力（自回归），主流方向 |
| Encoder-Decoder | T5、BART | 翻译/摘要 |

## 二、Tokenization

| 算法 | 原理 | 代表 |
|------|------|------|
| **BPE** | 从字符开始迭代合并最频繁字符对 | GPT |
| **SentencePiece** | BPE 变体，直接处理原始文本 | LLaMA、Gemini |
| **WordPiece** | 按概率合并 | BERT |

**关键概念**：
- Token ≠ 词：英文约 1 token ≈ 0.75 词，中文约 1 token ≈ 0.5 字
- 上下文窗口：模型 token 上限（GPT-4 128K、Claude 200K、Gemini 1M+）
- 成本：API 按 input + output token 计费

## 三、推理参数

| 参数 | 说明 | 建议 |
|------|------|------|
| **Temperature** | 随机性。0=确定，1=随机 | 代码 0，创意 0.7-0.9 |
| **Top-P** | 只从累积概率 top P 的 token 采样 | 0.9-1.0 |
| **Max Tokens** | 输出上限 | 根据任务设定 |
| **Stop Sequences** | 停止条件 | `\n\n`、`</output>` |

## 四、Prompt 工程

| 技巧 | 说明 |
|------|------|
| **Zero-shot** | 直接提问，无示例 |
| **Few-shot** | 提供 2-5 示例后再提问 |
| **Chain-of-Thought** | 要求逐步推理："Let's think step by step" |
| **角色设定** | 指定身份/语气 |
| **结构化输出** | 指定格式（JSON/Markdown） |

### System Prompt vs User Prompt

```
System: 你是精通 Java 的代码审查者。只关注安全漏洞和性能问题。
User:   审查以下代码：[code]
```

## 自测

1. **Self-Attention 中为什么要除以 √d_k？**
   <br/>→ d_k 较大时 QK^T 点积值变大，经 softmax 后梯度趋于 0（饱和区）。除以 √d_k 将方差控制在 1，缓解梯度消失问题。

2. **Decoder-only 架构为什么成为主流？Encoder-only 和 Encoder-Decoder 各适合什么任务？**
   <br/>→ Decoder-only 结构统一、训练效率高、自回归生成能力强（GPT/Claude/LLaMA）。Encoder-only（BERT）适合理解（分类/NER），Encoder-Decoder（T5）适合翻译/摘要。Decoder-only 的规模扩展性最好。

3. **Temperature = 0 和 Temperature = 1 的行为差异是什么？代码生成为什么建议用低温度？**
   <br/>→ T=0 总是选概率最高的 token（确定性输出），T=1 按原始概率采样（多样性高）。代码生成需要确定性和正确性，低温度减少随机性。创意写作需要高温度增加多样性。
