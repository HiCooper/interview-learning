## 一、HashMap

### 数据结构

HashMap 底层是**数组 + 链表 + 红黑树**的组合。

关键参数：
- 负载因子默认 **0.75**，容量默认 **16**

**put 流程**：
1. 计算 key 的 hashCode，找到对应 bin
2. bin 为 null → 直接存入
3. bin 不为空 → 遍历链表/红黑树，hashCode 相同更新 value，不同则插入尾部
4. 链表长度 **≥ 8** 且 Map 元素总数 **≥ 64** → 转为红黑树（O(n) → O(log n)）
5. 否则进行 `resize()` 扩容

HashMap 非线程安全，允许 null key/value，迭代器 fail-fast。

## 二、ConcurrentHashMap

### 线程安全实现

通过 **CAS + synchronized**：
- bin 节点为空 → CAS 尝试更新
- bin 节点非空 → synchronized 同步

### 对比

| 集合类 | 特点 |
|--------|------|
| **HashMap** | 非线程安全，允许 null，fail-fast |
| **HashTable** | 线程安全（全方法加锁），性能差 |
| **ConcurrentHashMap** | 线程安全（CAS+synchronized），高并发性能好，仅 bin 级别加锁 |
| **HashSet** | 基于 HashMap，不允许元素重复 |
| **LinkedHashMap** | 保证插入顺序 |
| **TreeMap** | 红黑树，SortedMap 接口，key 自然排序，O(log n) |

## 三、红黑树

### 定义

自平衡二叉查找树，通过颜色标记保证平衡。插入、删除、查找最坏时间复杂度均为 **O(log n)**。

统计性能好于 AVL 树。Java 中 HashMap、TreeMap、TreeSet 都有应用。

### 五个特性

1. 每个节点或者是**黑色**，或者是**红色**
2. **根节点**是黑色
3. 每个**叶子节点（NIL）**是黑色
4. 红色节点的**子节点必须是黑色**
5. 从一个节点到其子孙节点的所有路径上包含**相同数目的黑色节点**

若给定黑色节点数为 N：最短路径 = N 个连续黑色（高度 N-1），最长路径 = 红黑相间（高度 2(N-1)）。

### 树结构对比

| 树类型 | 特点 | 应用场景 |
|--------|------|----------|
| **红黑树** | O(log n)，少量数据增改有优势 | HashMap、TreeMap |
| **B 树** | 分支和叶子都存数据 | 文件索引 |
| **B+ 树** | 非叶子只存 key，数据全在叶子，减少磁盘 IO | MySQL 索引 |
| **AVL 树** | 严格平衡（高度差 ≤ 1） | 查询密集型 |

### B 树 vs B+ 树

| 维度 | B 树 | B+ 树 |
|------|------|-------|
| 关键字数量 | 分支节点 k-1 个 | 分支节点 M 个 |
| 数据存储 | 每个节点 | 仅叶子节点 |
| 查询 | 找到即返回 | 必须到叶子 |
| 分支存储 | 数据关键字 | 索引信息 |
