## 速查卡

- **快速排序**：选基准 → 分区 → 递归，平均 O(n log n)，最坏 O(n²)，不稳定
- **直接插入排序**：视第一个为有序，依次插入，O(n²)，稳定
- **希尔排序**：按间隔分组插入排序、逐步缩小间隔，平均 O(n log n)，不稳定
- **稳定性**：相等元素相对次序不变 → 稳定；交换后可能变 → 不稳定

## 一、快速排序

### 原理

1. 选择第一个数为基准值 p
2. 小于 p 的数放在左边，大于 p 的数放在右边
3. 递归对 p 左边和右边的子序列重复此过程

时间复杂度：O(n log n)

### Java 实现

```java
public static void quickSort(int[] a, int start, int end) {
    if (start < end) {
        int baseNum = a[start]; // 选基准值
        int midNum;
        int i = start;
        int j = end;
        do {
            while ((a[i] < baseNum) && i < end) { i++; }
            while ((a[j] > baseNum) && j > start) { j--; }
            if (i <= j) {
                midNum = a[i];
                a[i] = a[j];
                a[j] = midNum;
                i++;
                j--;
            }
        } while (i <= j);
        if (start < j) { quickSort(a, start, j); }
        if (end > i) { quickSort(a, i, end); }
    }
}
```

## 二、直接插入排序

### 原理

视第一个数为有序序列，依次取后面 n-1 个数与有序序列从后往前比较，找到比 insertNum 大的数则后移一位，找到位置后插入。

时间复杂度：O(n²)

### Java 实现

```java
public void insertSort(int[] a) {
    int len = a.length;
    int insertNum;
    for (int i = 1; i < len; i++) {
        insertNum = a[i];
        int j = i - 1;
        while (j >= 0 && a[j] > insertNum) {
            a[j + 1] = a[j];
            j--;
        }
        a[j + 1] = insertNum;
    }
}
```

## 三、希尔排序

### 原理

插入排序改进版，通过分组减少数据移动量：按间隔 len/2 分组，每组内插入排序，逐步缩小间隔。

### Java 实现

```java
public void shellSort(int[] a) {
    int len = a.length;
    while (len != 0) {
        len = len / 2;
        for (int i = 0; i < len; i++) {
            for (int j = i + len; j < a.length; j += len) {
                int k = j - len;
                int temp = a[j];
                while (k >= 0 && temp < a[k]) {
                    a[k + len] = a[k];
                    k -= len;
                }
                a[k + len] = temp;
            }
        }
    }
}
```

## 算法对比

| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定性 |
|------|:------:|:------:|:---:|:---:|
| 快速排序 | O(n log n) | O(n²) | O(log n) | 不稳定 |
| 直接插入 | O(n²) | O(n²) | O(1) | 稳定 |
| 希尔排序 | O(n log n) | O(n²) | O(1) | 不稳定 |

## 自测

1. **快速排序在什么情况下会退化为 O(n²)？如何避免？**
   <br/>→ 数组已有序（正序或逆序）时，每次只排出一个元素。避免：随机化选基准（Randomized QuickSort）、三数取中法、使用 `Arrays.sort()` 的双轴快排（Dual-Pivot QuickSort）。

2. **希尔排序和直接插入排序的核心区别是什么？为什么希尔排序更快？**
   <br/>→ 希尔排序先按大间隔分组排序，使元素「基本有序」后再缩小间隔，减少了最终插入排序时元素移动的距离。插入排序对基本有序的数组接近 O(n)。

3. **说说「稳定排序」的意义？HashMap 排序需要关心稳定性吗？**
   <br/>→ 稳定性：等值元素在排序后保持原相对顺序。多级排序时重要（先按 A 排序，再按 B 排序，需保持 A 的结果）。HashMap 无序，不保证任何顺序，需用 TreeMap（自然序）或 LinkedHashMap（插入序）。
