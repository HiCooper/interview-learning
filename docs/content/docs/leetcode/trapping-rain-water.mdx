## 问题描述

给定 `n` 个非负整数表示每个宽度为 1 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

**示例 1:**
- 输入: `height = [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]`
- 输出: `6`
- 解释: 蓝色部分表示雨水，总共 6 个单位

**示例 2:**
- 输入: `height = [4, 2, 0, 3, 2, 5]`
- 输出: `9`

**提示:**
- `n == height.length`
- `1 <= n <= 2 × 10⁴`
- `0 <= height[i] <= 10⁵`

## 速查卡

- **核心思路**：每个位置能接的水 = min(leftMax, rightMax) - height[i]
- **双指针维护**：leftMax 和 rightMax，谁小移动谁（min(leftMax, rightMax) 已确定）
- **时间复杂度**：O(n)，空间 O(1)
- **关联理论**：双指针 = 在有序性质上的空间压缩（左/右最大高度单调）

## 解法：双指针

### 思路

使用双指针 `left` 和 `right` 分别从数组两端向中间移动：

1. 维护 `leftMax` 和 `rightMax` 分别记录左边和右边的最大高度
2. 每次移动较矮一侧的指针：
   - 如果当前高度 ≥ 该侧最大高度，更新最大高度
   - 否则，当前柱子可以接到雨水：`water += maxHeight - height[current]`
3. 核心原理：**木桶效应** — 能接多少水由较矮的一侧决定

```mermaid
flowchart TD
    Start["left=0, right=n-1"] --> Loop{"left < right?"}
    Loop -->|是| Compare{"height[left] < height[right]?"}
    Compare -->|是| CheckLeft{"height[left] >= leftMax?"}
    CheckLeft -->|是| UpdateL["leftMax = height[left]"]
    CheckLeft -->|否| AddL["water += leftMax - height[left]"]
    UpdateL --> IncL["left++"]
    AddL --> IncL
    Compare -->|否| CheckRight{"height[right] >= rightMax?"}
    CheckRight -->|是| UpdateR["rightMax = height[right]"]
    CheckRight -->|否| AddR["water += rightMax - height[right]"]
    UpdateR --> IncR["right--"]
    AddR --> IncR
    IncL --> Loop
    IncR --> Loop
    Loop -->|否| End["返回 water"]
```

### 复杂度分析

| 指标 | 值 |
|------|-----|
| 时间复杂度 | O(n) — 每个元素访问一次 |
| 空间复杂度 | O(1) — 只用了 4 个变量 |

### Java 实现

```java
class Solution {
    public int trap(int[] height) {
        if (height == null || height.length == 0) {
            return 0;
        }

        int left = 0, right = height.length - 1;
        int leftMax = 0, rightMax = 0;
        int water = 0;

        while (left < right) {
            if (height[left] < height[right]) {
                if (height[left] >= leftMax) {
                    leftMax = height[left];
                } else {
                    water += leftMax - height[left];
                }
                left++;
            } else {
                if (height[right] >= rightMax) {
                    rightMax = height[right];
                } else {
                    water += rightMax - height[right];
                }
                right--;
            }
        }

        return water;
    }
}
```

## 关键点

- **双指针 vs 单调栈**: 双指针 O(1) 空间最优；单调栈解法 O(n) 空间但思路不同（按行累加）
- **移动策略**: 总是移动较矮一侧的指针，因为能接的水量由矮侧决定
- **左右对称**: 两侧逻辑完全对称，`leftMax` 和 `rightMax` 分别扮演左右边界角色
- **前缀/后缀最大值**: 另一种思路是用 `leftMax[i]` 和 `rightMax[i]` 前缀数组，但需要 O(n) 空间

---

## 自测

1. **接雨水双指针的核心思想是什么？为什么谁小移动谁？**
   <br/>→ 每个位置能接的水 = min(leftMax, rightMax) - height[i]。当 leftMax < rightMax 时，min(leftMax, rightMax) = leftMax 已被确定，无论 rightMax 后面多大都不影响当前位置。所以处理 left 侧并左移。反之同理。

2. **如果不用双指针，还有什么解法？各自的复杂度如何？**
   <br/>→ 暴力 O(n²)（每个位置向左右找最大）；前缀最大值数组 O(n) 时间 O(n) 空间（预处理 leftMax[] 和 rightMax[]）；单调栈 O(n) 时间 O(n) 空间（按行而非按列计算）。双指针是空间最优解 O(1)。
