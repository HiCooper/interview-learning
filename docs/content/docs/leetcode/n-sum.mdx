## 问题概述

N 数之和问题指在数组中查找 N 个元素，使其和等于目标值 `target`，并找出所有不重复的组合。

| 问题 | 解法 | 时间复杂度 |
|------|------|-----------|
| 2Sum | HashMap | O(n) |
| 3Sum | 排序 + 双指针 | O(n²) |
| 4Sum | 排序 + 固定前两个 + 双指针 | O(n²) |

---

## 速查卡

- **2Sum**：HashMap，遍历时查 target-nums[i]，O(n) 时间 O(n) 空间
- **3Sum**：排序 + 固定第一个 + 双指针左右向中间，O(n²) 时间
- **4Sum**：排序 + 固定前两个 + 双指针，O(n²)时间
- **去重**：排序后跳过与前一个相同的元素和指针
- **双指针模式**：`while (left < right)`，和<target→left++，和>target→right--

## 2Sum（两数之和）

### 问题描述

给定一个包含 `n` 个整数的数组 `nums` 和一个目标值 `target`，找出数组中两个元素 `a`, `b`，使得 `a + b = target`。返回其下标。

### 解法：HashMap

遍历数组，对于每个元素 `nums[i]`，检查 `target - nums[i]` 是否已在 HashMap 中。

```java
import java.util.HashMap;
import java.util.Map;

public class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> map = new HashMap<>();

        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i];
            if (map.containsKey(complement)) {
                return new int[]{map.get(complement), i};
            }
            map.put(nums[i], i);
        }

        return new int[0];
    }
}
```

**复杂度**: O(n) 时间，O(n) 空间。

## 3Sum（三数之和）

### 问题描述

给定一个包含 `n` 个整数的数组 `nums`，判断 `nums` 中是否存在三个元素 `a`, `b`, `c`，使得 `a + b + c = target`。找到所有满足条件且不重复的三元组。

### 解法：排序 + 双指针

1. 对数组排序
2. 遍历数组，对每个 `nums[i]`，在其后部分用双指针查找两数之和等于 `target - nums[i]`
3. 跳过重复元素避免重复组合

```java
import java.util.*;

public class Solution {
    public List<List<Integer>> threeSum(int[] nums, int target) {
        List<List<Integer>> result = new ArrayList<>();
        Arrays.sort(nums);

        for (int i = 0; i < nums.length - 2; i++) {
            if (i > 0 && nums[i] == nums[i - 1]) {
                continue; // 避免重复元素
            }
            int left = i + 1;
            int right = nums.length - 1;
            while (left < right) {
                int sum = nums[i] + nums[left] + nums[right];
                if (sum == target) {
                    result.add(Arrays.asList(nums[i], nums[left], nums[right]));
                    while (left < right && nums[left] == nums[left + 1]) {
                        left++;
                    }
                    while (left < right && nums[right] == nums[right - 1]) {
                        right--;
                    }
                    left++;
                    right--;
                } else if (sum < target) {
                    left++;
                } else {
                    right--;
                }
            }
        }

        return result;
    }
}
```

**复杂度**: O(n²) 时间，O(1) 额外空间（不计结果集）。

## 4Sum（四数之和）

### 问题描述

给定一个包含 `n` 个整数的数组 `nums` 和一个目标值 `target`，找出 `nums` 中所有可以使四个整数相加等于 `target` 的不重复组合。

### 解法：排序 + 固定前两个 + 双指针

1. 对数组排序
2. 双重循环固定前两个元素 `nums[i]` 和 `nums[j]`
3. 在剩余部分用双指针查找两数之和等于 `target - nums[i] - nums[j]`

```java
import java.util.*;

public class Solution {
    public List<List<Integer>> fourSum(int[] nums, int target) {
        List<List<Integer>> result = new ArrayList<>();
        Arrays.sort(nums);

        for (int i = 0; i < nums.length - 3; i++) {
            if (i > 0 && nums[i] == nums[i - 1]) {
                continue; // 避免重复元素
            }
            for (int j = i + 1; j < nums.length - 2; j++) {
                if (j > i + 1 && nums[j] == nums[j - 1]) {
                    continue; // 避免重复元素
                }
                int left = j + 1;
                int right = nums.length - 1;
                while (left < right) {
                    int sum = nums[i] + nums[j] + nums[left] + nums[right];
                    if (sum == target) {
                        result.add(Arrays.asList(
                            nums[i], nums[j], nums[left], nums[right]));
                        while (left < right && nums[left] == nums[left + 1]) {
                            left++;
                        }
                        while (left < right && nums[right] == nums[right - 1]) {
                            right--;
                        }
                        left++;
                        right--;
                    } else if (sum < target) {
                        left++;
                    } else {
                        right--;
                    }
                }
            }
        }

        return result;
    }
}
```

**复杂度**: O(n²) 时间，O(1) 额外空间。

## 关键点总结

- **2Sum**: HashMap 一次遍历，O(n)，空间换时间
- **3Sum/4Sum**: 排序是核心预处理，双指针在有序数组上 O(n) 查找两数之和
- **去重策略**: 
  - 固定元素去重：`if (i > 0 && nums[i] == nums[i-1]) continue`
  - 双指针去重：找到解后跳过重复的 `left` 和 `right`
- **泛化思路**: kSum 问题可递归降维为 (k-1)Sum，最终落到 2Sum（双指针或 HashMap）

## 自测

1. **2Sum 用 HashMap 为什么是 O(n)？如果用排序+双指针是 O(n log n)，为什么通常不用于 2Sum？**
   <br/>→ HashMap O(n)：一次遍历，每次 O(1) 查找。排序+双指针需要 O(n log n) 排序，且返回下标时排序会丢失原始索引。返回下标→HashMap，返回值→双指针也可以。

2. **3Sum 中去重为什么需要在固定元素和双指针两侧都做？**
   <br/>→ 固定元素去重：避免同一位置用相同值重复计算。双指针去重：找到一组解后，left 和 right 都可能指向重复值，需跳过避免重复解。三者缺一不可。

3. **4Sum 为什么能 O(n²) 而非 O(n³)？**
   <br/>→ 固定前两个 O(n²)，剩下两数用双指针 O(n) = O(n³)。但如果用 HashMap 预存所有两数之和（O(n²)），再查互补对（O(n²)），总体可到 O(n²)，但空间 O(n²)，需额外去重逻辑。
