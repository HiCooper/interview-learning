## 概述

通过 SSH 公钥认证，实现无需密码即可登录远程服务器。

## 速查卡

- **原理**：非对称加密。本地持私钥（保密），服务器存公钥（authorized_keys），连接时服务器用公钥加密挑战，客户端用私钥解密响应
- **配置流程**：ssh-keygen -t ed25519（生成密钥对，推荐Ed25519）→ ssh-copy-id -i ~/.ssh/id_ed25519.pub user@host（复制公钥到服务器）
- **算法选择**：Ed25519（现代推荐，更安全更快）> RSA 4096 > RSA 2048

## 配置步骤

### 1. 查看本地公钥

```bash
# Ed25519 算法（推荐）
cat ~/.ssh/id_ed25519.pub

# 或 RSA 算法
cat ~/.ssh/id_rsa.pub
```

### 2. 生成公钥（如果没有）

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

### 3. 复制公钥到服务器

**方法 A — 手动复制**：将公钥内容复制到剪贴板，粘贴到服务器的 `~/.ssh/authorized_keys` 文件中。

**方法 B — 一条命令**：

```bash
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server_ip
```

首次需要输入密码，之后就可以免密登录了。

## 原理

SSH 免密登录基于**非对称加密**：
- 本地持有**私钥**（保密，不传输）
- 服务器存储**公钥**（在 `authorized_keys` 中）
- 连接时服务器用公钥加密随机字符串，客户端用私钥解密并返回，验证身份

## 自测

1. **为什么推荐 Ed25519 而不是 RSA？**
   <br/>→ Ed25519 更安全（抗侧信道攻击）、密钥更短（256bit vs 2048+bit）、签名更快。RSA 需要更大密钥才能达到同等安全级别，且对量子计算更脆弱。

2. **SSH 免密登录的 authorized_keys 权限不正确会导致什么？**
   <br/>→ SSH 会拒绝使用该文件（安全策略）。正确权限：`~/.ssh/` 为 700（仅 owner 可读写执行），`~/.ssh/authorized_keys` 为 600（仅 owner 可读写）。权限过宽会导致 SSH 认为不安全而跳过公钥认证。
