## 速查卡

- **六大原则**：开闭（OCP，对扩展开放对修改关闭）→ 里氏代换（LSP）→ 依赖倒转（DIP）→ 接口隔离（ISP）→ 迪米特（LoD）→ 合成复用（CRP）
- **创建型**（5）：单例、工厂方法、抽象工厂、建造者、原型
- **结构型**（7）：适配器（改接口）、装饰器（增强功能）、代理（控制访问）、外观、桥接、组合、享元
- **行为型**（11）：策略、模板方法、观察者、迭代器、责任链、命令、备忘录、状态、访问者、中介者、解释器
- **代理 vs 适配器 vs 装饰器**：适配器改接口，代理控访问，装饰器增功能
- **模板方法 vs 策略**：模板方法用继承（骨架在父类），策略用组合（算法可替换）

## 一、六大设计原则

| 原则 | 核心思想 |
|------|----------|
| **开闭原则（OCP）** | 对扩展开放，对修改关闭（总纲） |
| **里氏代换（LSP）** | 基类可出现的地方，子类一定可以出现 |
| **依赖倒转（DIP）** | 针对接口编程，依赖抽象不依赖具体 |
| **接口隔离（ISP）** | 多个隔离的接口优于单个接口 |
| **迪米特法则（LoD）** | 最少知道原则，降低耦合 |
| **合成复用（CRP）** | 尽量使用合成/聚合，而非继承 |

## 三、常见设计模式

### 单例模式（Singleton）

保证一个类仅有一个实例，提供全局访问点。**构造函数私有**。

优点：减少内存开销、避免资源多重占用。
缺点：没有接口、不能继承。

使用场景：生产唯一序列号、WEB 计数器、创建对象消耗资源过多时。

### 模板方法（Template Method）

定义算法骨架，步骤延迟到子类。**在抽象类实现通用步骤**。

优点：封装不变、扩展可变、提取公共代码。
缺点：每个实现都需要子类。

一般加 `final` 防止恶意操作。

### 代理模式（Proxy）

为对象提供代理以控制访问。**实现与被代理类组合**。

适用：远程代理、虚拟代理、Copy-on-Write、保护代理、缓存代理、防火墙代理等。

与适配器区别：适配器改变接口，代理不改变。与装饰器区别：装饰器增强功能，代理加以控制。

### 装饰器模式（Decorator）

动态给对象添加额外职责。继承的替代方案，可动态扩展功能。

### 工厂模式（Factory Method）

定义创建对象接口，让子类决定实例化。扩展性高，屏蔽具体实现。

复杂对象适合工厂模式，简单对象直接 new。

### 抽象工厂（Abstract Factory）

创建一系列相关对象，保证客户端使用同一产品族。

产品族难扩展，产品等级易扩展。

### 适配器模式（Adapter）

将一个类的接口转换成客户希望的另一个接口。

优点：让无关类一起运行，提高复用和灵活性。
缺点：过多使用让系统零乱。

### 观察者模式（Observer）

定义一对多依赖关系。抽象类中有 ArrayList 存放观察者。

缺点：循环依赖可能导致崩溃。建议异步执行避免卡壳。

### 桥接模式（Bridge）

抽象与实现分离，各自独立变化。适用于存在两个独立变化维度的类。

## 自测

1. **代理模式和装饰器模式的核心区别？各举一个实际场景。**
   <br/>→ 代理控制访问（如延迟加载、权限检查），装饰器增强功能（如给 InputStream 加缓冲）。Spring AOP 是代理，Java IO 的 `BufferedInputStream` 是装饰器。

2. **模板方法模式和策略模式都用于封装算法，如何选择？**
   <br/>→ 模板方法用继承（算法骨架在父类，步骤由子类覆写），策略用组合（算法接口 + 具体实现类，运行时切换）。策略更灵活、符合组合优于继承原则；模板方法适合框架定流程。

3. **单例模式的双重检查锁定（DCL）为什么要用 volatile？**
   <br/>→ `instance = new Singleton()` 不是原子操作（分配内存→初始化→赋值），指令重排可能导致未初始化对象被其他线程读到。volatile 禁止指令重排，保证可见性。

4. **观察者模式和发布-订阅模式有区别吗？**
   <br/>→ 观察者：Subject 直接维护 Observer 列表，耦合度较高（同步通知）。发布-订阅：通过 Broker/EventBus 解耦，Publisher 和 Subscriber 互不知道对方存在（异步解耦）。MQ 是典型的发布-订阅。
