# 学习方法论：五维学习系统

> 知识库是「教材」，这五个方法是你的「练习册」。教材做得再好，不练 = 没学。

---

## 为什么需要这个系统？

传统学习路径只有一条：**读文档 → 读完 → 忘了 → 再读**。被动阅读的保留率仅 ~10%。

高效学习的核心原理来自认知科学：

| 原理 | 说明 |
|------|------|
| **主动回忆** | 大脑从"识别"切换到"生成"模式，神经通路完全不同 |
| **间隔重复** | 在遗忘临界点复习，每次复习间隔指数拉长（1天→3天→7天→30天） |
| **交错练习** | 混合不同主题比只刷一个主题长期 retention 高 43% |
| **有益的困难** | 先挣扎再学习，记忆痕迹更深 |
| **生成效应** | 说出来/写出来的东西比你"脑子里想了"记住得多得多 |

## 五种核心方法

### 一、问题前置（Problem-First）— 每次复习的第一步

**做法**：先不看答案，尝试自己回答。

```
错误方式：看自测题 → 瞄一眼答案 → "哦对我知道" → 其实没记住
正确方式：看自测题 → 拿纸/白板写答案 → 讲出来 → 再对照原文验证
```

**为什么有效**：大脑在搜索答案的过程中激活了相关神经通路。即使没答出来，这个"搜索"行为本身就在强化记忆。答错后再看正确答案，记忆效果远好于直接看答案。

**执行标准**：每次复习自测题时，先闭眼/看白墙，尝试在脑中组织答案，至少 30 秒后再看参考。

### 二、精加工提问（Elaborative Interrogation）— 追问三层"为什么"

**做法**：对每个核心概念连续追问"为什么"，直到触及底层原理。

```
表面记忆：
  "Redis 雪崩 → TTL 加随机值"

精加工链条：
  Q1: 为什么 TTL 加随机值能防雪崩？
  A1: 让 key 分散在不同时间过期，避免同时失效打向 DB

  Q2: 为什么是 ±30% 而不是 ±100% 或 ±200%？
  A2: 30% 已足够分散过期时间；加太多导致缓存驻留过久、数据更陈旧

  Q3: TTL 加随机值和多级缓存是什么关系？为什么不能只靠其中一种？
  A3: TTL 随机化防同时过期（预防性），多级缓存是已经大面积失效后的兜底（应急性）
       两者分工不同：一个降低概率，一个降低影响
```

**为什么有效**：孤立的事实容易遗忘，因果链条形成网络记忆。一旦理解底层逻辑，即使忘了"±30%"这个数字，也能从原理推导出来。

**执行标准**：每天至少对一个概念做 3 层"为什么"追问，把链条写在速查卡旁边。

### 三、交错练习（Interleaving）— 别只刷一个模块

**做法**：一个学习时段混合 2-3 个不同模块，而非只蹲在一个坑里。

```
分块学习（低效）：              交错学习（高效）：
周一：Java 并发 (1h)             周一：并发 20min + 缓存 20min + DP 20min
周二：缓存 (1h)                  周二：JVM 20min + MQ 20min + DP 20min
周三：DP 题 (1h)                 周三：集合 20min + Raft 20min + 接雨水 20min
```

**为什么有效**：

- 分块学习让大脑进入"惯性滑行"——不需要重新检索，感觉学得很顺但记得不牢
- 交错练习每次切换都要重新"加载"不同知识区，频繁的检索和关联极大强化记忆
- 真实面试就是交错模式：前一分钟问并发，下一分钟问缓存

**执行标准**：每天学习 40-60 分钟时，至少切换 2-3 个不同模块。具体轮换方案见 `review-schedule.md`。

### 四、费曼技巧（Feynman Technique）— 讲出来才是真懂

**做法**：拿一张白纸（或打开录音），假装给一个完全不懂的人讲这个概念。

```
步骤：
1. 选一个概念 → 在纸顶端写下概念名
2. 用最通俗的语言解释它 — 不能使用术语，不能跳过"显然"
3. 卡住的地方 → 标记 → 回去重新学 → 再讲
4. 简化 → 能不能用一个类比讲清楚？
```

**检验标准**：如果回听录音时你自己都觉得"挺清楚的"，那才是真掌握了。反之——"我当时在说什么？"——说明还要再学。

**为什么有效**：费曼强迫你把知识从内部表征（"我心里知道"）转换为外部表达（"别人能听明白"）。这个转换过程是学习行为中最深层的处理。

**执行标准**：每周至少 2 次，每次 1-2 个主题，每次 5-10 分钟，录音回听。

### 五、模拟面试（Mock Interview）— 压力下组织语言

**做法**：模拟真实面试场景，限时口头回答。

```
操作方式（用 AI）：
1. 告诉 Claude/GPT："你现在是面试官，从我的知识库里随机抽题
   （并发、缓存、DB、MQ、JVM），每题追问 1-2 层，限时 3 分钟"
2. 口头回答（不要打字！）
3. 回答完让 AI 点评：遗漏了什么？逻辑是否清晰？术语是否准确？
```

**和费曼的区别**：

| 维度 | 费曼技巧 | 模拟面试 |
|------|----------|----------|
| 目标 | 验证是否真懂 | 训练表达+抗压 |
| 方式 | 自己讲，不限时 | 被提问，限时回答 |
| 节奏 | 从容 | 有压力 |
| 反馈 | 自查盲点 | AI/人反馈质量 |

**为什么有效**：真实面试最大的挑战不是"不知道"，而是"在压力下组织不出连贯的回答"。模拟面试训练的就是这种"想了→说出来"的肌肉记忆。

**执行标准**：每周 3 次，每次 15 分钟（5 道题 × 3 分钟），口头回答并用手机录音。

## 日/周整合流程

### 每日学习模板（40 min）

```
 分钟  |  环节        |  内容
-------|-------------|--------------------------
 0-5   |  问题前置    | 从今天的 3 个模块各选
       |             | 1 道自测题，尝试默答
 5-10  |  速查回顾    | 读对应模块的速查卡
       |             | 验证刚才的自测答案
10-20  |  精加工提问  | 选 1 个没答上来的概念
       |             | 追问 3 层"为什么"
20-30  |  交错切换    | 切换到不同模块
       |             | 重复问题前置+速查回顾
30-40  |  费曼输出    | 选 1 个今天学的概念
       |             | 白纸讲解/录音 2 分钟
```

### 每周节奏

```
周一    周二    周三    周四    周五    周六        周日
────    ────    ────    ────    ────    ────        ────
日常    日常    日常    日常    日常    费曼 ×2      模拟面试 ×3
40min   40min   40min   40min   40min   选 2 个主题   AI 随机出题
        +模拟          +模拟          +录音回听      +复盘本周
        面试(15min)    面试(15min)                   +定下周计划
```

## 知识库在流程中的角色

```
            知识库文件
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
 速查卡      正文详解      自测题
 (2min)     (深入学习)    (检验)
    │           │           │
    │     ┌─────┘           │
    ▼     ▼                 ▼
 每日速查  精加工追问      问题前置的
 回顾环节  的参考材料      题目来源
```

- **速查卡** = 快速验证记忆（"我能不能从速查卡的 5 个点出发，展开讲 2 分钟？"）
- **正文** = 费曼讲解和精加工提问时的参考（"这个点讲不通 → 回头查正文 → 再讲"）
- **自测题** = 问题前置的题库 + 模拟面试的出题池
